Git - system for version controling

Git global configuration

Command line settings

git config --global user.name "meno"
git config --global user.email email@email.com
git config --global core.editor vim
git config --global merge.tool vimdiff

Config file $HOME/.gitconfig

[user]
    name = pklimo
    email = pklimo@gmail.com
[core]
    editor = vim
[merge]
    tool = vimdiff
[init]
    defaultBranch = main
[pull]
    rebase = false
[alias]
    hist = log --graph --abbrev-commit --decorate --format=format:'%C(bold blue)%h%C(reset) - %C(bold green)(%ar)%C(reset) %C(white)%s%C(reset) %C(dim white) %an%C(reset)%C(bold yellow)%d%C(reset)' --all

Access management

Password storage

  • git config credential.helper store password will be stored in default file ~/.git-credentials
  • git config credential.helper 'store --file=/home/user/.git-credentials' save into specified file

Login with ssh keys

  • generate ssh key: ssh-keygen -t rsa -b 4096 -C "pklimo@gmail.com"
  • add generated key into ssh agent ssh-add ~/.ssh/git_PKlimo_id_rsa
  • upload public key git_PKlimo_id_rsa.pub into github
  • test ssh connection ssh -T -p 443 git@ssh.github.com

Git repository configuration

command line settings

  • git remote add origin git@github.com:PKlimo/notes.git

config file .git/config

[core]
    repositoryformatversion = 0
    filemode = true
    bare = false
    logallrefupdates = true
[remote "origin"]
    url = git@github.com:PKlimo/notes.git
    fetch = +refs/heads/*:refs/remotes/origin/*
[branch "main"]
    remote = origin
    merge = refs/heads/main

pre-commit hook

  • before commit, git runs executable script .git/hooks/pre-commit
  • if script exit with return code different then zero, commit won’t be added to branch
  • symbolic link to python script /data/notes/system/python/pre-commit.py
    • test for size of file - max 500 000 B
    • test for execution bit
    • test for “privacy:secret” (two version - grep and committed files)
  • there are two approaches:
    • check only files, that are going to be committed - obey .gitignore files
    • check all files in directory - ignore .gitignore files - could be problem with virtual environments

  • gitignore files can be also nested in directories

Git commands

Information and status retrieval

  • git status changes from last commit (new - not tracked files, modified files, …)
  • git log show history of commits, better is to use alias git hist
  • git log -p subor show history of file as was changed by commits
  • git ls-tree -r main show tracked files

Working with git structure

branches

  • git branch -a list of all branches
  • git checkout -b new_branch create new local branch and switch to it
  • git branch new_branch create new branch, but remain in current one
  • git branch -m new_name rename local branch (current)
  • git branch -M new_name force rename local branch, also if branch new_name exists
  • git checkout master switch to master branch
  • git merge new_branch merge new_branch into current one (master)
  • git branch -d new_branch delete branch (after merging)

commits

  • git add -A add all files (not tracked, modified) to be committed in next commit (add files to index)
  • git commit -a -m "toto je komentar"
    • param a - commit all modified files - better is to add files by git add because of not tracked files
    • param m "komentar" - comment of commit
  • git reset HEAD^ --hard put the repository to state in previous commit, last commit is discarted
  • git reset HEAD^ discard the last commit, files are modified, necessary git restore to also discard change in files
  • git reset sha_commitu --hard restore files to state as they were after commit

repositories

  • git init create new repository in current directory
  • git pull get the actual version from remote (github) repository, update HEAD
  • git fetch download all commits from remote, don’t update HEAD in local repository
  • git push send new files (commit) back to remote (github) repository

Working with files in repository

  • git checkout HEAD^ change working directory into previous commit
